# Security Policy

## Supported Versions

Security fixes target the current `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for a security vulnerability.

Use GitHub private vulnerability reporting if it is enabled for the repository. If it is not enabled, contact the maintainer through the repository owner profile and include:

- A short description of the issue.
- Steps to reproduce.
- Affected commit, release, or APK.
- Any known impact.

LearnKana is fully local and does not use accounts, analytics, telemetry, remote storage, or a backend service. Security reports should still be handled privately when they involve data exposure, supply-chain risk, signing keys, or executable behavior.
